Learn HTML By Playing Around

Here's a secret: most computer programmers haven't formally studied the tools and languages they use all day. Instead, they learn by playing around. They try things out, make mistakes, and go down wrong pathways all the time. But the process is educational, engaging, and most of all fun.

Here's how you can learn the basics of web programming by just playing around.

Web programming consists of four basic technologies: HTML, CSS, a programming language, and a database. I suggest learning them in that order, since each one can build off the one before it.

The first step is to find a way to easily experiment with whatever you're doing. Most web programmers use a bewildering number of tools and applications to work — a text editor, the command line, a web server, and a browser. I recommend avoiding a lot of that hassle and instead starting with a service like Cloud9 IDE, which lets you program directly in your browser and not worry about all the extra pieces (and it's free!). You can — and should — learn about all the other pieces down the road, but don't need to immediately.

HTML is the code that browsers read to display web pages. Every web page you visit is (mostly) just a bunch of HTML behind the scenes. And the cool thing is that you can view that HTML easily in any browser. How you do it varies from browser to browser. I recommend using Google Chrome — it has lots of built-in tools that make playing around with HTML fun and easy. In Chrome, select View from the upper menu, then Developer, then View Source.

Now, many popular websites use very advanced techniques to make their pages load as quickly as possible. If you look at their source code, chances are it will just look like gibberish. Try finding some simpler sites and look at the HTML from those sites to get a feel for how HTML works.
